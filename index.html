Seatwork #1 OPERATION TRACKING

Reflection: I finally realized that Java's increment operators work differently depending on placement. With pre-increment (++i), the variable levels up before the math even starts. But with post-increment (i++), Java uses the old value first and saves the update for later. It's such a tiny detail, but it can absolutely wreck your logic if you're not paying attention.


Seatwork #2 ATM TRANSACTION COUNTER

Explanation: For this seatwork, I developed a Java-based ATM Transaction Counter. The system manages an initial balance of 8,000 pesos and uses a while loop to keep the menu running until the user chooses to exit. I implemented logic to handle deposits, withdrawals with a 2,000-peso limit, and a counter to track total transactions processed during the session.

Reflection: This seatwork was a great exercise in state management and input validation. Working through the transaction counter specifically helped me visualize how Java handles increment timing. I learned that small details, like whether a counter updates before or after a process, are what make a system truly reliable and user-friendly.

Activity #1 VARIABLES

Explanation: This activity looks at how Java handles memory by splitting different variables between the Stack and the Heap. It shows how "aliasing" works, where multiple variables point to the same spot in memory, so changing one updates the other automatically.

Reflection: Seeing how the Stack and Heap are separate helped me realize that reference variables are just pointers to data, not the actual data. It was a big lightbulb moment to learn that copying an object just copies its memory address, which explains why variables can stay "in sync" during a program.

Activity #3 SIMPLE WALLET SYSTEM

Explanation: This activity is a Java program that helps manage money by letting users check their balance, add cash, or spend it within set limits. It uses a while loop to keep the menu open and has strict checks to block any spending over 1,000 pesos or what's currently in the wallet.

Reflection: Building this taught me about defensive programming by using nested logic to catch mistakes before they update the balance. I realized that clear error messages are just as important as the actual code because they help the user fix their input without the whole program crashing.

SimpleWalletA_Jose.java
1import java.util.Scanner;

12 System.out.println("\n===== WALLET MENU =====");
13 System.out.println("1. View Balance");
14 System.out.println("2. Add Money");
15 System.out.println("3. Spend Money");
16 System.out.println("4. Exit");
17 System.out.print("Enter choice: ");
18 input = ans.nextInt();
19 switch (input) {
20 case 1:
21 tCount++;
22 System.out.println("Balance: " + initialBal + " pesos");
23 break;
24 case 2:
25 System.out.print("Add: ");
26 double addAmount = ans.nextDouble();
27 initialBal += addAmount;
28 tCount++;
29 System.out.println("New Balance: " + initialBal + " pesos");

31 case 3:
32 System.out.print("Spend: ");

Activity #4 GRADE EVALUATION SYSTEM

Explanation: This Java program automatically calculates a student's average grade from their Prelim, Midterm, and Final scores. It uses an if-else if ladder to give specific remarks like "Failed" or "Excellent" depending on which numerical bracket the average falls into.

Reflection: This activity showed me how to turn a school grading rubric into actual logic using boundary conditions. I also learned that using System.out.printf is a must for formatting averages so the output looks clean and professional for the user.

GradeEvaluationA_Jose.java
1public class GradeEvaluationA_Jose {
2 public static void main(String[] args) {
3 double prelim = 85;
4 double midterm = 78;
5 double finals = 90;
6 double avrgGrade = (prelim + midterm + finals) / 3;
7 System.out.println("=================================================");
8 System.out.println("|| Welcome to A_Jose's Grade Evaluation System ||");
9 System.out.println("=================================================");
10 System.out.println("\nPrelim Grade: " + prelim);
11 System.out.println("Midterm Grade: " + midterm);
12 System.out.println("Final Grade: " + finals);
13 System.out.printf("\nAverage: %.2f%n", avrgGrade);
14 String remark = "";
15 if (avrgGrade < 75) {
16 remark = "Failed";
17 } else if (avrgGrade >= 75 && avrgGrade <= 79) {
18 remark = "Fair";
19 } else if (avrgGrade >= 80 && avrgGrade <= 89) {
20 remark = "Good";
21 } else if (avrgGrade >= 90 && avrgGrade <= 100) {
22 remark = "Excellent";
23 } else {
24 System.out.println("Invalid Average Grade.");

Activity #5 PERSONAL EXPENSE TRACKER

Explanation: This Java program tracks spending in categories like food and clothing to see if it fits within a set budget. It uses static methods like totalExp and budgetStatus to handle the math and give a quick summary of how much money is left.

Reflection: Building this taught me how to keep code organized by breaking it down into smaller, modular parts. I realized that using conditional logic to compare expenses against a budget is a great way to solve real-world money management problems with code.

ExpenseTrackerA_Jose.java
1import java.util.Scanner;
2public class ExpenseTrackerA_Jose {
3 static void progTitle() {
4 System.out.println("===========================================");
5 System.out.println("|| Welcome to A_Jose's Expense Tracker ||");
6 System.out.println("===========================================");
7 }
8 static double totalExp(double food, double transport, double clothing, double other) {
9 return food + transport + clothing + other;
10 }
11 static String budgetStatus(double total, double budget) {
12 double remaining = budget - total;
13 if (total > budget) {
14 return "Over budget by: PHP " + String.format("%.2f", Math.abs(remaining));
15 } else {
16 return "Remaining budget PHP " + String.format("%.2f", remaining);
17 }
18 }
19 static void summary(double food, double transport, double clothing, double other, double budget) {
20 double total = totalExp(food, transport, clothing, other);
21 String status = budgetStatus(total, budget);
22 System.out.println("\n-------------------------------------------");
23 System.out.println("\nTOTAL: PHP " + total);
24 System.out.println("BUDGET: PHP " + budget);
25 System.out.println("STATUS: " + status);
26 if (total > budget) {
27 System.out.println("\nYou exceeded your budget.");
28 } else {
29 System.out.println("\nYou're within your budget.");
30 }
31 }
32 public static void main(String[] args) {
33 Scanner ans = new Scanner(System.in);
34 double food, transport, clothing, other, budget;
35 progTitle();
36 System.out.print("\nEnter food expenses: ");
37 food = ans.nextDouble();
38 while (food < 0) {
39 System.out.print("Please input a valid amount: ");
40 food = ans.nextDouble();
41 }
42 System.out.print("Enter transportation expenses: ");
43 transport = ans.nextDouble();
44 while (transport < 0) {
45 System.out.print("Please input a valid amount: ");
46 transport = ans.nextDouble();
47 }
48 System.out.print("Enter clothing expenses: ");
49 clothing = ans.nextDouble();
50 while (clothing < 0) {
51 System.out.print("Please input a valid amount: ");
52 clothing = ans.nextDouble();
53 }
54 System.out.print("Enter other expenses: ");
55 other = ans.nextDouble();
56 while (other < 0) {
57 System.out.print("Please input a valid amount: ");
58 other = ans.nextDouble();
59 }
60 System.out.print("Enter your budget: ");
61 budget = ans.nextDouble();
62 while (budget < 0) {
63 System.out.print("Please input a valid amount: ");
64 budget = ans.nextDouble();

67 }
68}
